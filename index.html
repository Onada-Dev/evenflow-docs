Feedback System

New in v2

A learning loop that improves search results over time based on user feedback.

How It Works

Hover over any product card to reveal feedback buttons. Thumbs up is instant. Thumbs down opens a comment box asking "Why was this a poor result?" Feedback is fire-and-forget — no loading spinners, no interruption to your workflow.

1
Search
User enters a natural language query
2

3
Learn
Score recalculated via Laplace smoothing
4
Improve
Future searches boosted/penalized (0.85x–1.2x)

Key Improvements

  • Hybrid search combines vector similarity with BM25 text search via Reciprocal Rank Fusion (RRF)
  • Cohere Rerank 3.5 provides ML-based semantic reranking of results (replaces basic attribute matching)
  • Cohere embed-v4 embeddings (1024d) — higher MTEB score than OpenAI text-embedding-3
  • Vendor tier system — Top 5 vendors boosted 1.3x, Top 10 boosted 1.2x, Top 20 boosted 1.1x
  • Category priority system — per-category vendor boost/exclusion across 12 product categories
  • Feedback loop — thumbs up/down with free-text comments, Laplace-smoothed scoring, precomputed boost
  • Vendor diversity filter — max 3 products per vendor to ensure variety in results
  • Removed 10 deleted vendors per client request (Artesia, Christopher Guy, Duralee, Thibaut, Maxwell, Tomlinson, Peyton Webster, Castelle, Wesley Hall, Noir)

Remaining Work

Data Quality Fixes

  1. Theodore Alexander (Top 5, 2,507 products) — Re-scrape to capture descriptions
  2. Precedent Furniture (Top 5, 406 products) — Re-scrape to capture image URLs
  3. Essentials for Living (Top 10, 408 products) — Re-scrape to capture images and longer descriptions
  4. Chelsea House (1,944 products) — Scrape descriptions
  5. Merge "Sarreid" + "Sarreid Ltd" — Deduplicate vendor entries (4 + 599 products)

Search Quality Improvements

  1. Contextual enrichment — Run LLM-generated descriptions for products with NO-DESC. Dramatically improves embedding quality.
  2. Re-embed after enrichment — Products with better descriptions produce better embeddings
  3. Query intent extraction — Have Claude identify primary item type to avoid keyword pollution (e.g. "ottoman" in console table query)
